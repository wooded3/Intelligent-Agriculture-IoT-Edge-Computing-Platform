# 项目架构快速参考卡片

## 🎯 30秒快速介绍

"我们项目采用**前后端分离架构**，前端用Vue 3，后端用Spring Boot，数据库用MySQL。系统分为5个核心模块：设备管理、数据管理、告警管理、自动化控制、系统维护。前后端通过HTTP/JSON通信。"

---

## 📐 架构三句话

1. **架构模式**：前后端分离，三层架构（前端层、后端层、数据层）
2. **技术栈**：Vue 3 + Spring Boot + MySQL + MyBatis
3. **核心特点**：模块化设计，松耦合，易扩展

---

## 🏗️ 架构层次（四层）

```
设备层（IoT设备）
    ↓ HTTP协议
网络通信层
    ↓
平台服务层（Spring Boot后端）
    ├─ 设备管理模块
    ├─ 数据管理模块
    ├─ 告警管理模块
    ├─ 自动化控制模块
    └─ 系统维护模块
    ↓ MyBatis
数据层（MySQL数据库）
    ↑
应用层（Vue 3前端）
```

---

## 🔧 技术选型理由（一句话）

| 技术 | 一句话理由 |
|------|-----------|
| Spring Boot | 快速开发，生态成熟，易于集成 |
| Vue 3 | 学习曲线平缓，性能优秀，TypeScript支持 |
| MySQL | 成熟稳定，性能优秀，成本低 |
| MyBatis | SQL可控性强，学习成本低，性能优化方便 |

---

## 📦 五大模块职责（一句话）

1. **设备管理**：设备的注册、查询、更新、删除
2. **数据管理**：接收设备数据，存储和查询
3. **告警管理**：规则配置，自动告警检测
4. **自动化控制**：自动化规则配置和管理
5. **系统维护**：教室信息管理，系统状态检查

---

## 🔄 数据流转（五步）

```
设备上报 → 后端接收 → 数据存储 → 告警检测 → 前端展示
```

## ⚠️ 常见问题快速回答

**Q: 为什么前后端分离？**
A: 开发效率高、技术栈灵活、易于维护、可扩展性强

**Q: 为什么不用微服务？**
A: 项目规模小、团队规模小、业务简单，单体架构足够

**Q: 如何保证数据一致性？**
A: 使用@Transactional事务、参数验证、异常处理、数据库约束

**Q: 如何扩展？**
A: 水平扩展（多实例+负载均衡）、数据库优化、缓存、消息队列
