# 小组实验总结（Group Summary）

## 实验实际完成情况

本项目为"智慧教室物联网边缘计算平台"，旨在构建一个基于物联网技术的智能监控与自动化控制系统。系统通过实时监控教室环境（如温度、湿度、光照、空气质量等），实现智能告警、自动化控制和数据分析，从而优化教学环境、提升能源利用效率并改善师生体验。

根据项目规划，我们采用了前后端分离架构，前端使用Vue 3 + TypeScript + Vite + Element Plus，后端使用Spring Boot + MyBatis + MySQL。系统采用分层架构设计，分为设备层、网络通信层、平台服务层、应用层，并按业务域划分为5个核心模块。

在功能实现方面，我们完成了设备管理、数据管理、告警管理、自动化控制、系统维护等核心功能模块。设备管理模块实现了设备的注册、查询、更新、删除、统计等完整功能，支持多维度筛选。数据管理模块实现了数据接收、存储、查询功能，支持按设备ID和时间范围检索，并提供分页功能。告警管理模块实现了告警规则配置、自动检测、告警处理等完整流程。自动化控制模块支持自动化规则的创建、启用/禁用、删除等操作。系统维护模块提供了教室信息管理和系统状态检查功能。前端实现了6个完整页面，包括仪表板、设备管理、数据监控、告警中心、自动化控制、系统管理，使用Element Plus组件库构建了美观流畅的用户界面，并使用ECharts实现了数据可视化。

数据库设计方面，我们创建了8个完整的数据表，包括设备表、传感器数据表、告警规则表、活跃告警表、告警历史表、自动化规则表、控制指令表、教室信息表。所有表均包含完整的字段定义、索引、外键约束，并提供了示例数据。API接口方面，我们实现了完整的RESTful API接口，所有接口遵循统一的响应格式。

## 与预期的比较

根据项目规划，我们计划分为4个迭代（Sprint）。Sprint 1完成了需求分析、技术选型、架构设计和数据库概念设计，创建了UML模型。Sprint 2完成了设备管理和数据采集基础功能，并超出预期提前实现了告警管理的基础功能。Sprint 3完成了自动化规则和数据可视化，并超出预期实现了完整的告警管理模块和所有前端页面。Sprint 4完成了系统功能测试、前后端集成测试和数据库功能测试，但Docker容器化和云平台部署尚未实现。

从功能完成度来看，设备管理、数据管理、告警管理、自动化控制、系统维护、前端界面、数据可视化、数据库设计等模块均达到100%完成度，总体完成度约95%。技术实现方面，前端技术栈已完整实现，后端技术栈中Spring Boot、MyBatis、MySQL、RESTful API均已实现，但Redis和RabbitMQ未实现。数据库方面已完整实现MySQL数据库配置、表结构、索引优化、外键约束和示例数据。

## 项目亮点与收获

本项目采用前后端分离架构和模块化设计，实现了数据持久化存储，使用Element Plus构建了响应式UI，使用ECharts实现了数据可视化。功能上实现了完整的设备管理流程、智能告警系统、自动化控制和数据查询分析。代码质量方面遵循编码规范，添加了完善的注释，实现了统一的错误处理和RESTful接口设计。

在开发过程中，我们遇到了前后端集成、数据库设计、路由冲突、前端页面跳转等技术难点，通过配置CORS、引入MySQL和MyBatis、调整路由顺序、重构为Vue Router单页应用等方式解决了这些问题。在开发流程方面，我们采用敏捷开发模式应对需求变更，优先实现核心功能。

通过本次项目，我们掌握了全栈开发能力，包括前端Vue 3开发、后端Spring Boot开发、数据库设计等。学会了如何设计系统架构、划分模块、设计数据库，提高了问题解决能力和团队协作能力。在项目管理方面，实践了敏捷开发模式，学会了规划迭代、管理任务，提高了文档编写能力和测试能力。

## 不足与改进

当前系统还存在一些不足：测试覆盖不足，主要关注功能测试，单元测试和集成测试覆盖不足；性能优化方面，数据量大时可能存在性能问题；部署方案仅提供了本地部署，Docker容器化和云平台部署尚未实现；安全性方面缺少用户认证和授权机制；实时性方面，数据更新需要手动刷新。

后续改进计划包括：功能完善方面，增加用户认证与授权、使用WebSocket实现实时数据推送、支持数据导出、开发移动端适配；技术优化方面，优化数据库查询和增加缓存机制、重构代码提高质量、增加单元测试和集成测试；部署优化方面，实现Docker容器化、建立CI/CD流程、实现云平台部署。

## 总结

本次实验项目"智慧教室物联网边缘计算平台"基本按照计划完成，实现了设备管理、数据管理、告警管理、自动化控制、系统维护等核心功能模块，前后端功能完整，数据库设计合理，系统可以正常运行。虽然部分计划功能（如Docker容器化、云平台部署）尚未实现，但核心功能已全部完成，系统可以满足基本的业务需求。通过本次项目，我们不仅掌握了全栈开发技术，还提高了团队协作能力和项目管理能力。未来，我们将继续完善系统功能，优化系统性能，提高代码质量，使系统更加完善和实用。
