# 《软件工程实验》实验指导书

**软件工程课题组**  
**二零二五**

---

## 一、课程目标

1. 使学生能够按照特定的软件选题调研分析相关软件的系统需求，并应用UML建立软件分析模型。
2. 使学生能够结合数据库、现代Web开发框架及API设计等相关知识，设计并实现能够满足问题描述和约束条件的软件系统解决方案；能够应用UML等建模工具建立软件设计模型，并依据**敏捷开发**理念进行系统原型开发与迭代。
3. 通过实验分组安排，使学生具有通过团队配合进行有效的分工，并能协调合作共同完成软件项目目标的能力。
4. 通过规范的实验报告撰写要求，使学生了软件科技文档的书写要求，并能够准确规范的撰写分析设计文档。

---

## 二、实验环境

### 1. 建模工具
- StarUML、Rational Rose 或 PlantUML 等其他 UML 建模工具

### 2. 协作与版本控制
- **Git** + **GitHub** / **GitLab** / **Gitee**

### 3. 开发工具（可选，鼓励使用新技术栈）
- **前端**：Vue.js / React / Angular / Bootstrap
- **后端**：Spring Boot (Java) / Express.js (Node.js) / Django (Python) / .NET Core (C#)
- **数据库**：MySQL / MongoDB
- **文档工具**：Microsoft Office / WPS Office / Markdown
- **部署**：鼓励使用 Docker 容器化，并可部署至云平台（如阿里云、腾讯云、Heroku）的试用套餐

---

## 三、实验要求

1. 4-5 人自愿结为一个项目小组，每组指定一名组长，必须使用 Git 进行版本控制，并有清晰的 Commit 记录，以体现协作过程。
2. 每组从实验题目中任选一题，也可由小组经协商并经指导教师确认符合实验要求后确定自选题目。
3. 采用敏捷开发模式，以 1-2 周为一个迭代周期（Sprint），每个迭代结束时需提交可工作的软件增量。鼓励使用看板（如 GitHub Projects）进行任务管理。
4. 系统建模要求采用 UML 建模方法，鼓励绘制**用户故事地图**辅助需求分析。
5. 每个小组提交一份完整的实验报告（提交打印和电子两种形式），文档格式、内容参照附件实验报告样本。
6. 组长需在报告中提供基于 Git 提交记录和任务完成情况的组内互评表，作为评分重要依据。

---

## 四、实验内容

### 实验一：需求分析与迭代规划
按照特定的软件选题，调研分析、建立用户故事地图、确定软件的系统需求，并应用 UML 建立软件问题模型。

### 实验二：系统设计与技术选型
应用软件设计知识设计出能够满足问题描述和约束条件的软件系统解决方案，并应用 UML 建立软件设计模型。

### 实验三：系统原型构建与实现
应用适当的软件开发工具进行软件原型搭建，并选取部分模块编码实现。

### 实验四：软件测试与部署
设计软件测试方案，并对已实现的系统模块设计测试用例，实施测试，分析测试结果并写出部署方案。

---

## 五、实验项目选题

### 物联网边缘计算平台（智慧农业/智慧教室场景）

**总体描述**：构建一个基于物联网技术的智能监控与自动化控制系统，可在智慧农业或智慧教室场景中任选其一。项目组需要实现设备管理、数据采集、智能告警与自动化控制闭环。

**【分析与设计重点】**：分析设备管理、数据采集、自动化规则等核心需求；设计“设备-数据-规则”模型，并使用活动图描述复杂的自动化控制逻辑；在架构设计上，建议采用分层架构，并区分边缘设备层、网络通信层、平台服务层与应用层，用部署图进行描述。

**【实现与测试要求】**：实现设备数据接收、存储与可视化展示；配置并触发至少一条自动化控制规则；测试数据流的完整性与规则触发的准确性。
